# Guia de Contribucion

## Flujo de trabajo
1. Crear rama desde dev.
2. Mantener cambios pequenos y descriptivos.
3. Enviar Pull Request hacia dev.
4. Pasar revisiones antes de merge.

## Estandares
1. Documentar cambios en Markdown tecnico.
2. Mantener coherencia con el alcance: laboratorio local, sin contenedores.
3. No introducir dependencias de nube publica.

## Commits
1. Usar mensajes claros en espanol o ingles tecnico.
2. Ejemplo: feat(terraform): agrega variables de red interna
