# 00. Resumen Ejecutivo

## Contexto
Este proyecto propone un marco de seguridad informatica para una arquitectura cliente-servidor en un laboratorio local simulado. El objetivo es demostrar, de forma academica y tecnica, que un conjunto de politicas bien definidas puede reducir riesgos operativos y de ciberseguridad en entornos de pequena y mediana escala.

## Propuesta
Se define una arquitectura minima con un servidor Linux y dos clientes sobre red interna simulada. Sobre esta base se implementan controles de acceso, autenticacion, gestion de usuarios, cifrado en transito, proteccion de datos, auditoria, actualizaciones, respaldos y respuesta ante incidentes.

## Enfoque metodologico
1. Planeacion y delimitacion de alcance.
2. Diseno de laboratorio virtual local.
3. Aprovisionamiento base con Terraform cuando sea viable.
4. Configuracion tecnica con scripts Bash y PowerShell.
5. Redaccion formal de politicas.
6. Simulaciones controladas y recopilacion de evidencia.
7. Analisis de resultados y recomendaciones.

## Resultado esperado
Una solucion reproducible, defendible y facil de explicar, con trazabilidad completa entre riesgos, politicas, controles implementados y pruebas realizadas.
